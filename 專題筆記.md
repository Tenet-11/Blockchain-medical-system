# Flask 與 API 基礎筆記

## 1. 什麼是 API？

`API` 全名是：

* `Application Programming Interface`
* 應用程式介面

這個名字比較抽象，可以先記成：

* `API` = 讓不同程式互相溝通的接口

### 用餐廳比喻

假設：

* 客人 = 瀏覽器 / `Unity`
* 服務生 = `API`
* 廚房 = 後端程式 / `Python` 程式

你不能直接衝進廚房拿食材，而是要透過服務生點餐。

普通文字範例：

客人 → 服務生/API → 廚房/後端程式
瀏覽器/Unity → API → Python 程式

你跟服務生說：

* 我要查詢所有病歷

服務生把請求送進廚房，廚房處理完再把結果送回來。

所以在你的專題裡：

* `API` = 讓外部可以呼叫你的醫療紀錄系統的入口

---

## 2. 你的 API 實際長什麼樣？

你現在有這種網址：

* `http://127.0.0.1:5000/records`

這就是一個 `API`。

當你在瀏覽器輸入它，瀏覽器會對你的 `Flask` 程式說：

* 我要取得所有病歷資料

然後你的 `Python` 程式回傳：

普通文字範例：

[
{
"record_id": "R001",
"patient_id": "P001",
"diagnosis": "Flu"
}
]

這種回傳格式叫做 `JSON`。

---

## 3. 什麼是 JSON？

`JSON` 是一種資料格式，很像 `Python` 的 `dictionary`。

### Python dictionary

普通文字範例：

{
"record_id": "R001",
"diagnosis": "Flu"
}

### JSON

普通文字範例：

{
"record_id": "R001",
"diagnosis": "Flu"
}

它的用途是：

* 讓不同程式都看得懂資料

例如：

* `Python` 看得懂 `JSON`
* `JavaScript` 看得懂 `JSON`
* `Unity C#` 也可以讀 `JSON`
* 手機 App 也可以讀 `JSON`

所以 `API` 常常會回傳 `JSON`。

---

## 4. 什麼是 Flask？

`Flask` 是 `Python` 的一個套件。

可以先記成：

* `Flask` = 用 `Python` 建立網站或 `API` 的工具

你原本的 `Python function` 只能在 `Python` 裡面被呼叫。

例如：

普通文字範例：

def get_records():
records = get_all_records()
return records

這個 `function` 本來只能在程式裡面呼叫：

普通文字範例：

get_records()

但是 `Flask` 可以讓這個 `function` 變成一個網址：

普通文字範例：

@app.route("/records")
def get_records():
...

變成：

* `http://127.0.0.1:5000/records`

也就是：

* `Flask` 把 `Python function` 變成 `API` 網址

這句很重要。

---

## 5. 你的 Flask 程式在做什麼？

看這段：

普通文字範例：

from flask import Flask, request, jsonify

意思是從 `Flask` 套件拿三個工具：

* `Flask` → 建立伺服器
* `request` → 接收使用者傳來的資料
* `jsonify` → 把 `Python` 資料轉成 `JSON` 回傳

### `app = Flask(__name__)`

普通文字範例：

app = Flask(**name**)

意思是：

* 建立一個 `Flask` 應用程式

你可以把 `app` 想成：

* 你的 `API` 伺服器本體

後面所有的 `API` 都是掛在這個 `app` 上。

---

## 6. 什麼是伺服器？

伺服器就是：

* 一直開著，等待別人連線的程式

一般 `Python` 程式通常是：

* 執行 → 做完事情 → 結束

例如：

普通文字範例：

print("hello")

它印完就結束。

但 `Flask` 不一樣。

你執行：

普通文字範例：

app.run(debug=True)

它會開始等待請求：

* 有人連 `/records` 嗎？
* 有人連 `/verify/R002` 嗎？
* 有人連 `/tamper/R002` 嗎？

所以它會一直跑，直到你按：

* `Ctrl + C`

把它關掉。

---

## 7. 什麼是 127.0.0.1:5000？

你看到這個：

* `http://127.0.0.1:5000`

可以拆成三部分：

* `http://` → 使用 `HTTP` 協定
* `127.0.0.1` → 你自己的電腦
* `5000` → `Flask` 預設的門牌號碼，也叫 `port`

所以：

* `127.0.0.1` = `localhost` = 我自己的電腦

你的 `API` 現在不是公開在網路上，而是開在你自己的電腦裡。

所以別人不能直接連，只有你自己可以用瀏覽器打：

* `http://127.0.0.1:5000/records`
