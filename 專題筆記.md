# Step 3：鏈下資料庫建置（SQLite）

## 1. Step 3 的目標

Step 3 的核心是建立一個 `SQLite` 資料庫，用來儲存完整的醫療紀錄資料。

在這個專題中，我們採用的是：

```text
Off-chain storage + Blockchain verification
```

也就是：

```text
完整病歷資料 → 存在 SQLite 資料庫
資料 hash / 區塊資訊 → 用區塊鏈概念驗證完整性
```

因為醫療資料通常內容較多，不適合全部直接放在區塊鏈上，所以使用 `SQLite` 作為鏈下資料庫。

---

## 2. 為什麼要使用 SQLite？

`SQLite` 是一種輕量型資料庫。

它的特點是：

* 不需要額外安裝資料庫伺服器
* 會直接產生一個 `.db` 檔案
* 適合小型專題、demo、原型系統
* 可以用 `DB Browser for SQLite` 直接查看資料表內容

在本專題中，資料庫檔案名稱為：

```python
DB_NAME = "medical_records.db"
```

這代表程式會建立或使用一個叫做：

```text
medical_records.db
```

的資料庫檔案。

---

## 3. 初始化資料庫：`init_db()`

程式中使用 `init_db()` 來建立資料庫與資料表。

```python
def init_db():
    conn = sqlite3.connect(DB_NAME)
    cursor = conn.cursor()

    cursor.execute("""
    CREATE TABLE IF NOT EXISTS medical_records (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        record_id TEXT NOT NULL UNIQUE,
        patient_id TEXT NOT NULL,
        doctor_name TEXT NOT NULL,
        diagnosis TEXT NOT NULL,
        prescription TEXT NOT NULL,
        timestamp TEXT NOT NULL,
        record_hash TEXT NOT NULL
    )
    """)

    conn.commit()
    conn.close()
```

這段程式的意思是：

* 連接 `medical_records.db`
* 如果資料庫不存在，SQLite 會自動建立
* 建立一張 `medical_records` 資料表
* 如果資料表已經存在，就不會重複建立

---

## 4. medical_records 資料表欄位

你的資料表結構如下：

| 欄位名稱           | 功能              |
| -------------- | --------------- |
| `id`           | 資料庫內部流水號        |
| `record_id`    | 病歷編號，例如 `R001`  |
| `patient_id`   | 病患編號，例如 `P001`  |
| `doctor_name`  | 醫師姓名            |
| `diagnosis`    | 診斷結果            |
| `prescription` | 處方內容            |
| `timestamp`    | 病歷建立時間          |
| `record_hash`  | 該筆病歷資料計算出的 hash |

其中比較重要的是：

```text
record_id TEXT NOT NULL UNIQUE
```

這代表：

* `record_id` 不能是空的
* `record_id` 不能重複

例如已經有 `R001`，就不能再新增另一筆 `R001`。

---

## 5. 插入資料：`insert_record(record)`

`insert_record()` 負責把一筆 `MedicalRecord` 物件存進 SQLite。

```python
def insert_record(record):
    conn = sqlite3.connect(DB_NAME)
    cursor = conn.cursor()

    try:
        cursor.execute("""
        INSERT INTO medical_records
        (record_id, patient_id, doctor_name, diagnosis, prescription, timestamp, record_hash)
        VALUES (?, ?, ?, ?, ?, ?, ?)
        """, (
            record.record_id,
            record.patient_id,
            record.doctor_name,
            record.diagnosis,
            record.prescription,
            record.timestamp,
            record.calculate_hash()
        ))

        conn.commit()
        print(f"[DB] Record {record.record_id} inserted.")

    except sqlite3.IntegrityError:
        print(f"[DB] Record {record.record_id} already exists.")

    conn.close()
```

這裡做了幾件事：

* 將病歷資料存進 `medical_records`
* 同時把該筆病歷的 `record_hash` 一起存進資料庫
* 如果 `record_id` 重複，會進入 `except sqlite3.IntegrityError`
* 避免程式因為重複新增資料而直接崩潰

---

## 6. 為什麼 INSERT 裡面用 `?`

這段：

```python
VALUES (?, ?, ?, ?, ?, ?, ?)
```

代表用參數方式插入資料。

不要直接把字串拼進 SQL，因為可能有安全問題。

例如不推薦：

```python
cursor.execute("INSERT INTO medical_records VALUES (" + record.record_id + ")")
```

比較好的寫法是：

```python
cursor.execute("""
INSERT INTO medical_records (...)
VALUES (?, ?, ?, ?, ?, ?, ?)
""", (...))
```

這樣可以避免 SQL injection，也比較安全。

## 7. 查詢全部資料：`get_all_records()`

這個 function 用來從 SQLite 讀取所有病歷。

```python
def get_all_records():
    conn = sqlite3.connect(DB_NAME)
    cursor = conn.cursor()

    cursor.execute("""
    SELECT record_id, patient_id, doctor_name, diagnosis, prescription, timestamp, record_hash
    FROM medical_records

    rows = cursor.fetchall()
    conn.close()

```

重點：

* `fetchall()` 代表把查詢到的所有資料都取出來
* 回傳結果是 `list` 裡面包很多筆 `tuple`

例如可能長這樣：

# Step 4：完整性驗證機制

## 1. Step 4 的目標

Step 4 的核心是：

```text
檢查資料有沒有被竄改
```

方法是：

```text
從資料庫讀取病歷
重新計算 hash
與原本儲存的 hash 比對
如果相同 → VALID
如果不同 → INVALID
```

這個機制可以偵測鏈下資料庫中的病歷資料是否被惡意修改。

---

## 2. MedicalRecord 類別

在驗證之前，先看 `MedicalRecord` 這個 class。

```python
class MedicalRecord:
    def __init__(self, record_id, patient_id, doctor_name, diagnosis, prescription, timestamp=None):
        self.record_id = record_id
        self.patient_id = patient_id
        self.doctor_name = doctor_name
        self.diagnosis = diagnosis
        self.prescription = prescription
        
        if timestamp:
            self.timestamp = timestamp
        else:
            self.timestamp = datetime.now().strftime("%Y-%m-%d %H:%M:%S")
```

`MedicalRecord` 用來表示一筆病歷資料。

每筆病歷包含：

* `record_id`
* `patient_id`
* `doctor_name`
* `diagnosis`
* `prescription`
* `timestamp`

如果建立物件時沒有給 `timestamp`，程式會自動產生目前時間。

---

## 3. 將病歷轉成 dictionary：`to_dict()`

```python
def to_dict(self):
    return {
        "record_id": self.record_id,
        "patient_id": self.patient_id,
        "doctor_name": self.doctor_name,
        "diagnosis": self.diagnosis,
        "prescription": self.prescription,
        "timestamp": self.timestamp
    }
```

這個 function 會把 `MedicalRecord` 物件轉成 `dictionary`。

原因是後面要用 `json.dumps()` 轉成固定格式的字串，再進行 hash 計算。

---

## 4. 計算病歷 hash：`calculate_hash()`

```python
def calculate_hash(self):
    record_string = json.dumps(self.to_dict(), sort_keys=True, ensure_ascii=False)
    return hashlib.sha256(record_string.encode()).hexdigest()
```

這是完整性驗證的核心。

流程是：

```text
MedicalRecord 物件
→ to_dict()
→ json.dumps()
→ encode()
→ sha256()
→ hexdigest()
→ 得到 hash 字串
```

也就是把病歷內容轉成一串固定長度的 hash。

只要病歷內容有任何改變，例如：

```text
Cold → Cancer
```

重新計算出來的 hash 就會完全不同。

---

## 5. 為什麼要用 `sort_keys=True`？

```python
json.dumps(self.to_dict(), sort_keys=True, ensure_ascii=False)
```

`sort_keys=True` 的意思是：

```text
把 dictionary 的 key 固定排序
```

因為 hash 很敏感，只要字串順序不同，hash 就會不同。

例如這兩個資料意思一樣：

```python
{"record_id": "R001", "diagnosis": "Flu"}
```

```python
{"diagnosis": "Flu", "record_id": "R001"}
```

但如果轉成字串的順序不同，hash 可能就會不同。

所以使用 `sort_keys=True` 可以讓 hash 計算更穩定。

---

## 6. Block 類別

`Block` 代表區塊鏈中的一個區塊。

```python
class Block:
    def __init__(self, index, medical_record, previous_hash):
        self.index = index
        self.timestamp = datetime.now().strftime("%Y-%m-%d %H:%M:%S")
        self.medical_record = medical_record
        self.previous_hash = previous_hash
        self.hash = self.calculate_hash()
```

每個 block 包含：

* `index`：區塊編號
* `timestamp`：區塊建立時間
* `medical_record`：該區塊儲存的病歷資料
* `previous_hash`：前一個區塊的 hash
* `hash`：目前區塊自己的 hash

---

## 7. Block hash 的計算

```python
def calculate_hash(self):
    block_string = json.dumps({
        "index": self.index,
        "timestamp": self.timestamp,
        "medical_record": self.medical_record.to_dict() if isinstance(self.medical_record, MedicalRecord) else self.medical_record,
        "previous_hash": self.previous_hash
    }, sort_keys=True, ensure_ascii=False)
    return hashlib.sha256(block_string.encode()).hexdigest()
```

這裡計算的是整個 block 的 hash。

它會把以下資料一起算進去：

* 區塊編號
* 區塊時間
* 醫療紀錄內容
* 前一個區塊的 hash

所以如果其中任何一項被改動，block hash 就會改變。

---

## 8. Blockchain 類別

```python
class Blockchain:
    def __init__(self):
        self.chain = [self.create_genesis_block()]
```

`Blockchain` 裡面有一個 `chain` list，用來存放所有 block。

一開始會先建立：

```text
Genesis Block
```

也就是創世區塊。

---

## 9. 創世區塊：`create_genesis_block()`

```python
def create_genesis_block(self):
    genesis_record = "Genesis Block"
    return Block(0, genesis_record, "0")
```

創世區塊是整條鏈的第一個區塊。

它沒有前一個區塊，所以 `previous_hash` 設為：

```text
0
```

---

## 10. 新增區塊：`add_block()`

```python
def add_block(self, medical_record):
    latest_block = self.get_latest_block()

    new_block = Block(
        index=len(self.chain),
        medical_record=medical_record,
        previous_hash=latest_block.hash
    )
    self.chain.append(new_block)
```

新增區塊時會：

* 找到目前最後一個 block
* 取得最後一個 block 的 hash
* 建立新的 block
* 新 block 的 `previous_hash` 指向上一個 block 的 hash
* 把新 block 加到 chain 裡

這就是區塊鏈「一個接一個串起來」的概念。

---

## 11. 驗證整條鏈：`is_chain_valid()`

```python
def is_chain_valid(self):
    for i in range(1, len(self.chain)):
        current_block = self.chain[i]
        previous_block = self.chain[i - 1]

        if current_block.hash != current_block.calculate_hash():
            print(f"Block {i} hash is invalid!")
            return False

        if current_block.previous_hash != previous_block.hash:
            print(f"Block {i} previous hash is invalid!")
            return False

    return True
```

這個 function 會檢查兩件事：

### 第一件事：目前區塊 hash 是否正確

```python
if current_block.hash != current_block.calculate_hash():
```

意思是：

```text
目前存的 hash
是否等於重新計算出來的 hash
```

如果不同，代表這個 block 的內容可能被改過。

---

### 第二件事：previous_hash 是否正確連到上一個區塊

```python
if current_block.previous_hash != previous_block.hash:
```

意思是：

```text
目前區塊記錄的 previous_hash
是否等於上一個區塊真正的 hash
```

如果不同，代表鏈的連結被破壞。

---

## 12. 驗證單一區塊：`verify_block()`

```python
def verify_block(self, block_index):
    ...
```

這個 function 用來驗證某一個 block 是否有效。

它會：

* 取得指定 block
* 重新計算該 block 的 hash
* 比對原本儲存的 block hash
* 如果一樣，輸出 `VALID`
* 如果不同，輸出 `INVALID`

重點程式：

```python
stored_block_hash = block.hash
recalculated_block_hash = block.calculate_hash()
```

意思是：

```text
stored_block_hash = 原本儲存在 block 裡的 hash
recalculated_block_hash = 根據目前資料重新計算出來的 hash
```

如果：

```text
stored_block_hash == recalculated_block_hash
```

代表資料沒有被改。

如果：

```text
stored_block_hash != recalculated_block_hash
```

代表資料可能被竄改。

## 13. 資料庫完整性驗證：`verify_record_from_db()`

這是你 Step 4 demo 最重要的 function。

```python
def verify_record_from_db(record_id, blockchain=None):
    row = get_record_by_id(record_id)

    if not row:
        print("Record not found in database.")
        return False

    db_record = MedicalRecord(
        record_id=row[0],
        patient_id=row[1],
        doctor_name=row[2],
        diagnosis=row[3],
        prescription=row[4],
        timestamp=row[5]
    )

    recalculated_hash = db_record.calculate_hash()
    stored_hash = row[6]
```

這裡的流程是：

```text
1. 從 SQLite 找出指定 record_id 的病歷
2. 用資料庫讀出的內容重新建立 MedicalRecord 物件
3. 根據目前資料重新計算 hash
4. 取出資料庫中原本儲存的 hash
5. 比對兩個 hash 是否相同
```

---

## 14. 為什麼要重新建立 MedicalRecord？

資料庫讀出來的是 `tuple`，例如：

```python
("R002", "P002", "Dr. Lin", "Cold", "Medicine B", "2026-05-03 21:29:13", "hash...")
```

但 `calculate_hash()` 是 `MedicalRecord` class 裡的方法。

所以要先用資料庫資料重建：

```python
db_record = MedicalRecord(...)
```

然後才能呼叫：

```python
db_record.calculate_hash()
```

## 16. 模擬資料竄改：`tamper_database_record()`

```python
def tamper_database_record(record_id, new_diagnosis):
    conn = sqlite3.connect(DB_NAME)
    cursor = conn.cursor()

    cursor.execute("""
    UPDATE medical_records
    SET diagnosis = ?
    WHERE record_id = ?
    """, (new_diagnosis, record_id))

    conn.commit()
    conn.close()

    print(f"[DB] Record {record_id} diagnosis changed to {new_diagnosis}.")
```

這個 function 用來模擬資料被竄改。

例如：

```python
tamper_database_record("R002", "Cancer")
```

會把 R002 的 diagnosis 從：

```text
Cold
```

改成：

```text
Cancer
```

但是注意：

```text
它只改 diagnosis
沒有更新 record_hash
```

所以資料內容改了，但原本的 hash 還是舊的。

這就是為什麼再次驗證時會得到：

```text
INVALID
```

## 18. Step 4 的核心概念

Step 4 的重點不是阻止資料被修改，而是：

```text
就算資料被修改，系統也能偵測出來
```

也就是：

```text
資料被竄改
→ 重新計算 hash
→ 與原本 hash 不一致
→ 判定 INVALID
```

這就是完整性驗證。

## 19. Step 3 和 Step 4 的關係

Step 3 和 Step 4 是連在一起的。

```text
Step 3：建立 SQLite 資料庫
Step 4：根據 SQLite 中的資料進行完整性驗證
```

Step 3 負責：

* 儲存完整醫療資料
* 儲存原始 `record_hash`

Step 4 負責：

* 從資料庫讀取病歷
* 重新計算 hash
* 與原始 hash 比對
* 判斷資料是否被竄改

所以整體流程是：

```text
新增病歷
→ 計算 hash
→ 存入 SQLite
→ 之後讀取資料
→ 重新計算 hash
→ 比對 hash
→ 判斷 VALID / INVALID
```

# STEP 5

## 1. 什麼是 API？

`API` 全名是：

* `Application Programming Interface`
* 應用程式介面

這個名字比較抽象，可以先記成：

* `API` = 讓不同程式互相溝通的接口

### 用餐廳比喻

假設：

* 客人 = 瀏覽器 / `Unity`
* 服務生 = `API`
* 廚房 = 後端程式 / `Python` 程式

你不能直接衝進廚房拿食材，而是要透過服務生點餐。

普通文字範例：

客人 → 服務生/API → 廚房/後端程式
瀏覽器/Unity → API → Python 程式

你跟服務生說：

* 我要查詢所有病歷

服務生把請求送進廚房，廚房處理完再把結果送回來。

所以在你的專題裡：

* `API` = 讓外部可以呼叫你的醫療紀錄系統的入口

---

## 2. 你的 API 實際長什麼樣？

你現在有這種網址：

* `http://127.0.0.1:5000/records`

這就是一個 `API`。

當你在瀏覽器輸入它，瀏覽器會對你的 `Flask` 程式說：

* 我要取得所有病歷資料

然後你的 `Python` 程式回傳：

普通文字範例：

[
{
"record_id": "R001",
"patient_id": "P001",
"diagnosis": "Flu"
}
]

這種回傳格式叫做 `JSON`。

---

## 3. 什麼是 JSON？

`JSON` 是一種資料格式，很像 `Python` 的 `dictionary`。

### Python dictionary

普通文字範例：

{
"record_id": "R001",
"diagnosis": "Flu"
}

### JSON

普通文字範例：

{
"record_id": "R001",
"diagnosis": "Flu"
}

它的用途是：

* 讓不同程式都看得懂資料

例如：

* `Python` 看得懂 `JSON`
* `JavaScript` 看得懂 `JSON`
* `Unity C#` 也可以讀 `JSON`
* 手機 App 也可以讀 `JSON`

所以 `API` 常常會回傳 `JSON`。

---

## 4. 什麼是 Flask？

`Flask` 是 `Python` 的一個套件。

可以先記成：

* `Flask` = 用 `Python` 建立網站或 `API` 的工具

你原本的 `Python function` 只能在 `Python` 裡面被呼叫。

例如：

普通文字範例：

```python
def get_records():
records = get_all_records()
return records
```

這個 `function` 本來只能在程式裡面呼叫：

普通文字範例：

get_records()

但是 `Flask` 可以讓這個 `function` 變成一個網址：

普通文字範例：
```python
@app.route("/records")
def get_records():
...
```

變成：

* `http://127.0.0.1:5000/records`

也就是：

* `Flask` 把 `Python function` 變成 `API` 網址

這句很重要。

---

## 5. 你的 Flask 程式在做什麼？

看這段：

普通文字範例：

from flask import Flask, request, jsonify

意思是從 `Flask` 套件拿三個工具：

* `Flask` → 建立伺服器
* `request` → 接收使用者傳來的資料
* `jsonify` → 把 `Python` 資料轉成 `JSON` 回傳

### `app = Flask(__name__)`

普通文字範例：

app = Flask(**name**)

意思是：

* 建立一個 `Flask` 應用程式

你可以把 `app` 想成：

* 你的 `API` 伺服器本體

後面所有的 `API` 都是掛在這個 `app` 上。

---

## 6. 什麼是伺服器？

伺服器就是：

* 一直開著，等待別人連線的程式

一般 `Python` 程式通常是：

* 執行 → 做完事情 → 結束

例如：

普通文字範例：

print("hello")

它印完就結束。

但 `Flask` 不一樣。

你執行：

普通文字範例：

app.run(debug=True)

它會開始等待請求：

* 有人連 `/records` 嗎？
* 有人連 `/verify/R002` 嗎？
* 有人連 `/tamper/R002` 嗎？

所以它會一直跑，直到你按：

* `Ctrl + C`

把它關掉。

---

## 7. 什麼是 127.0.0.1:5000？

你看到這個：

* `http://127.0.0.1:5000`

可以拆成三部分：

* `http://` → 使用 `HTTP` 協定
* `127.0.0.1` → 你自己的電腦
* `5000` → `Flask` 預設的門牌號碼，也叫 `port`

所以：

* `127.0.0.1` = `localhost` = 我自己的電腦

你的 `API` 現在不是公開在網路上，而是開在你自己的電腦裡。

所以別人不能直接連，只有你自己可以用瀏覽器打：

* `http://127.0.0.1:5000/records`


## 8. 什麼是 GET 和 POST？

`GET` 和 `POST` 都是 `HTTP` 請求方法。

可以先記成：

* `GET` → 用來「取得資料」
* `POST` → 用來「送出資料 / 新增資料 / 修改資料」

---

## 9. GET 是什麼？

`GET` 通常用在「查詢資料」。

例如你在瀏覽器輸入：

* `http://127.0.0.1:5000/records`

這其實就是送出一個 `GET` 請求。

意思是：

* 我要取得所有病歷資料

在你的程式中：

```
@app.route("/records", methods=["GET"])
def get_records():
    records = get_all_records()
    return jsonify(records)
```

這段代表：

* 當有人用 `GET` 連到 `/records`
* Flask 就會執行 `get_records()`
* 然後回傳所有病歷資料

所以：

* `GET /records` → 查詢全部病歷
* `GET /records/R002` → 查詢 `R002` 這筆病歷
* `GET /verify/R002` → 驗證 `R002` 是否有效

---

## 10. POST 是什麼？

`POST` 通常用在「送出資料」。

例如新增一筆病歷時，不能只靠網址，還需要把病歷內容一起送給後端。

例如要新增：

```
{
  "record_id": "R004",
  "patient_id": "P004",
  "doctor_name": "Dr. Lee",
  "diagnosis": "Headache",
  "prescription": "Medicine D"
}
```

這些資料會透過 `POST` 送到 Flask。

在你的程式中：

```
@app.route("/records", methods=["POST"])
def add_record():
    data = request.json
```

這段代表：

* 當有人用 `POST` 連到 `/records`
* Flask 會透過 `request.json` 取得對方送來的資料
* 再把資料新增到資料庫和區塊鏈中

所以：

* `POST /records` → 新增一筆病歷

---

## 11. GET 和 POST 的差別

| 比較項目         | `GET`   | `POST`              |
| ------------ | ------- | ------------------- |
| 主要用途         | 查詢資料    | 新增或送出資料             |
| 資料放哪裡        | 通常放在網址中 | 通常放在 request body 中 |
| 能不能用瀏覽器網址列測試 | 可以      | 不方便                 |
| 是否會改變資料      | 通常不會    | 通常會                 |
| 例子           | 查詢病歷    | 新增病歷、修改資料           |

簡單記法：

* `GET` = 我要拿資料
* `POST` = 我要送資料給後端處理

---

## 12. 為什麼瀏覽器網址列通常只能測 GET？

當你直接在瀏覽器網址列輸入：

* `http://127.0.0.1:5000/records`

瀏覽器預設會送出 `GET` 請求。

所以查資料很方便。

但如果是 `POST`，通常需要額外帶資料，例如：

```
{
  "record_id": "R004",
  "patient_id": "P004"
}
```

網址列沒有辦法很方便地放這些資料，所以通常會用：

* `Postman`
* `Thunder Client`
* 前端表單
* JavaScript `fetch()`
* Unity 的 HTTP request

來送出 `POST` 請求。
